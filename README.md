# 企业级私有知识库系统

## 项目概述

这是一个企业级的私有知识库问答系统，采用RAG（检索增强生成）架构，具备BM25缓存、混合检索、完整安全控制和微服务架构。系统已集成多项企业级优化功能，包括先进的文档解析、智能分块策略、高效检索算法和全面的安全防护。

## 架构特性

### 1. 微服务架构
- **API网关**：统一入口、路由和负载均衡
- **认证服务**：JWT认证和RBAC权限控制
- **QA服务**：核心问答逻辑处理
- **向量存储服务**：**Milvus企业级向量数据库**（默认），支持ChromaDB（备选）
- **BM25缓存服务**：多级缓存机制
- **文档处理服务**：文档解析和索引

### 2. 安全特性
- **JWT认证**：安全的令牌管理
- **RBAC权限控制**：基于角色的访问控制
- **速率限制**：令牌桶算法防止API滥用
- **熔断机制**：防止级联故障
- **数据加密**：敏感信息保护
- **权限过滤**：检索结果按用户权限过滤

### 3. 性能优化
- **多级缓存**：内存→Redis→持久化
- **异步处理**：高并发支持
- **向量化加速**：使用Milvus实现快速语义检索
- **混合检索**：向量+BM25关键词结合
- **RRF融合算法**：Reciprocal Rank Fusion优化检索结果排序
- **BGE重排序**：使用BGE-reranker模型提升重排序精度
- **智能路由**：FastText轻量模型实现查询类型智能分类

### 4. 企业级文档处理
- **多格式支持**：PDF、PPT、DOCX、XLS、CSV、图像等
- **深度解析**：集成Unstructured、pdfplumber、PaddleOCR
- **表格提取**：自动识别和提取文档中的表格数据
- **内容清洗**：自动清理页眉页脚、乱码、冗余空格
- **父子分块**：Parent-Child Chunking策略，平衡语义完整性与检索精度
- **增量入库**：基于文件MD5+修改时间戳的增量更新
- **去重机制**：余弦相似度去重，过滤低质文本

### 5. 可扩展性（企业级）
- **Milvus向量数据库**：支持大规模数据集、分布式部署、高级查询功能
- **水平扩展**：支持多节点部署和负载分发
- **高可用性**：内置复制和故障转移机制
- **元数据索引**：向量库建立标量索引支持复杂查询

### 6. 可观测性
- **结构化日志**：统一日志格式
- **性能指标**：Prometheus监控
- **健康检查**：服务状态监控
- **错误追踪**：全面错误处理
- **审计日志**：记录用户查询、文档操作

### 7. LLM容灾机制
- **多模型支持**：主用Ollama本地模型，备用阿里云/通义千问API
- **自动降级**：超时自动切换到备用模型
- **健康检查**：持续监控LLM服务状态

## 技术栈

- **Web框架**: FastAPI
- **向量存储**: **Milvus** (企业级首选), ChromaDB (备选)
- **缓存**: Redis, 多级缓存
- **安全**: JWT, bcrypt, python-jose
- **异步处理**: asyncio, aiohttp
- **监控**: Prometheus, structlog, loguru
- **文档解析**: Unstructured, pdfplumber, pytesseract, PaddleOCR
- **检索算法**: BM25, Sentence Transformers, BGE-reranker
- **容器化**: Docker, Docker Compose

## 优化亮点

### 数据处理层优化
1. **文档解析优化**：集成Unstructured + pdfplumber + PaddleOCR，支持PDF、Word、PPT、图片、扫描件解析，重点实现表格提取；解析后用正则清洗页眉页脚、乱码、冗余空格
2. **分块优化**：实现父子分块（Parent-Child Chunking），结合标题层级+Sentence-BERT语义相似度切分，避免语义割裂
3. **入库优化**：基于文件MD5+修改时间戳实现增量入库，入库前用余弦相似度去重，过滤低质文本；给每个文本块添加元数据（业务分类、权限等级、更新时间、标签），并在向量库建立标量索引

### 检索层优化
1. **混合检索实现**：完善BM25 + BGE-M3混合检索，用RRF（Reciprocal Rank Fusion）算法融合两路检索结果，避免简单拼接
2. **重排序优化**：加入bge-reranker-base模型，对召回的Top20结果进行二次精排，最终返回Top5
3. **向量库与缓存优化**：替换单机FAISS为Milvus分布式向量库；用Redis替换本地内存缓存，设置TTL自动过期，支持分布式高并发
4. **权限过滤**：在检索逻辑中添加权限过滤，根据用户角色筛选可访问的文档内容

### RAG核心层优化
1. **查询路由**：用FastText轻量模型实现智能路由，精准区分通用知识和专业知识，避免关键词判断的误判
2. **Prompt优化**：设计结构化模板（包含角色定义、业务约束、Few-shot示例），支持配置化管理（可在线修改、灰度、回滚）
3. **幻觉校验**：添加LLM自检逻辑，确保生成的回答完全来自知识库，拒绝编造内容
4. **多轮对话优化**：实现对话历史自动摘要和Token动态裁剪，防止上下文窗口溢出
5. **LLM容灾**：实现多LLM主备容灾（主用Ollama本地模型，备用阿里云/通义千问API），超时自动降级

### API服务层优化
1. **安全优化**：添加JWT身份认证和RBAC权限系统，实现用户登录鉴权和角色权限分配
2. **并发优化**：用Redis实现令牌桶限流，防止高并发攻击；完善FastAPI的SSE标准流式输出
3. **文档优化**：自动生成Swagger/ReDoc接口文档，规范接口对接

### 工程化与可观测性优化
1. **容器化**：编写Dockerfile，实现项目容器化部署，支持K8s水平扩缩容
2. **监控日志**：集成Prometheus + Grafana，监控QPS、P99延迟、召回率、LLM调用成功率；输出JSON结构化日志，集成ELK存储，添加审计日志（记录用户查询、文档操作）
3. **备份与测试**：实现向量库+源文件每日增量备份；编写Pytest单元测试和接口测试，保证代码稳定性
